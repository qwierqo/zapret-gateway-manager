# Архитектура проекта

## Цель

Сделать лёгкий manager поверх `zapret` для Linux-шлюза, через который проходит трафик всей домашней сети.

## Основные сущности

### 1. Profiles
Файлы профилей описывают:
- имя профиля;
- назначение;
- аргументы для запуска;
- набор проверок.

### 2. Apply layer
Модуль применения профиля:
- читает выбранный профиль;
- пишет нужную конфигурацию;
- запускает или перезапускает рабочий режим.

### 3. Checks layer
Проверяет доступность сервисов и записывает результат.

### 4. State layer
Хранит:
- последний выбранный профиль;
- последний рабочий профиль;
- служебные логи.

### 5. UI layer
Консольное меню для запуска основных режимов.

## Принцип развития

Проект развивается от простого к сложному:
1. каркас;
2. ручное применение профиля;
3. health-check;
4. автоподбор;
5. install script.
